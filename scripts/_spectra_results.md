## Q70 [G] — testuser1 @ Core AI Protocol Development

**Prompt:** How does the Task Aging feature work, and where do I configure its thresholds?

**query_type:** `help`  |  **elapsed:** 3.4s

**Spectra's answer:**

The **Task Aging** feature provides an adaptive visual indicator on kanban cards that tracks how long a task has remained in its current column. The badge updates automatically, escalating from a neutral grey to an amber **warning**, and finally to a red **critical** status as the task stalls. The count resets whenever a task is moved to a new column.

You can configure these thresholds in two ways:

*   **Board-level:** You can set global thresholds for the entire board within **Board Settings**.
*   **Column-level:** You can customize thresholds for specific columns by clicking the **⋮ menu** on the column header and selecting **Aging Alerts**. From there, you can choose to inherit the board settings, set custom day counts for warning/critical levels, or disable the feature for that specific column. 

Note that aging alerts are disabled by default for "Done" or "Backlog" style columns.
